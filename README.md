# Learning to Coordinate Multi-LLM Rationales for Multi-Trait Essay Scoring

Existing automated essay scoring methods that use large language models to generate multi-trait rationales rely solely on a single LLM's generation capability or fixed fusion strategies, making it difficult to achieve flexible expert fusion based on the characteristics of different essays. This paper proposes an adaptive expert fusion (\textbf{AEF}) method that employs an adaptive gating mechanism to adaptively integrate rationales generated by different LLMs, dynamically adjusting the contribution of each expert based on the representation of the reasoning basis. In addition, to address the problem of the gating biasing towards a single expert in multi-expert collaboration, we designed a joint coordination loss that combines a generation loss with an expert load balancing loss. This loss helps preserve the participation of both heterogeneous experts while maintaining scoring accuracy. Experiments on the ASAP/ASAP++ and Feedback Prize benchmark datasets show that our method significantly outperforms existing baselines in terms of Quadratic Weighted Kappa (QWK), validating the effectiveness of adaptive expert fusion in multi-trait automated essay scoring.This provides more effective support for the interpretability of essay scoring and the accuracy of multi-feature dimension scoring.
## Overview

Existing rationale-enhanced essay scoring methods usually rely on a single LLM rationale or fixed fusion strategies. AEF addresses this limitation by adaptively coordinating rationales from different LLM experts according to each essay and its corresponding rationale representations.

The framework mainly includes:

* Multi-LLM rationale generation using GPT and Llama.
* Adaptive expert fusion with sample-specific gating weights.
* Load-balancing auxiliary regularization to reduce one-sided expert preference.

## Model Architecture

AEF follows an encoder-decoder scoring framework. The essay and the rationales generated by GPT and Llama are encoded into shared representations. Then, an adaptive gating mechanism computes the fusion weights for different expert rationales. The fused representation is passed to the decoder to generate scores for multiple traits sequentially.

The training objective mainly includes:

* Generation loss for score sequence prediction.
* Load-balancing auxiliary loss for preventing the adaptive gate from over-relying on a single expert.
* Optional auxiliary trait-score supervision for stabilizing representation learning.

## Datasets

The experiments are conducted on:

* ASAP/ASAP++
* Feedback Prize English Language Learning

Please download the datasets from their official sources and organize them under the `data/` directory.

## Requirements

Install the required packages with:

```bash
pip install -r requirements.txt
```

## Local Model Files

The code loads pretrained models from local paths. Please place the pretrained backbone models under:

```text
models/
├── t5-base/
├── flan-t5-base/
├── bart-base/
└── pegasus/
```

You can modify the model path in the code if your local directory is different.

## Training

Train AEF on the ASAP/ASAP++ dataset with T5:

```bash
CUDA_VISIBLE_DEVICES=0 python main.py \
  --gpu 0 \
  --use_adaptive_rmts \
  --data asap \
  --model_name t5-base \
```

Train AEF on the Feedback Prize dataset:

```bash
CUDA_VISIBLE_DEVICES=0 python main.py \
  --gpu 0 \
  --use_adaptive_rmts \
  --data feedback \
  --model_name t5-base \
```

## Fixed-Weight Fusion Ablation

To evaluate fixed expert weights, use:

```bash
CUDA_VISIBLE_DEVICES=0 python main.py \
  --gpu 0 \
  --use_adaptive_rmts \
  --fixed_equal_gate \
  --gpt_weight 0.5 \
  --llama_weight 0.5 \
  --data asap \
  --model_name t5-base \
```

You can change `--gpt_weight` and `--llama_weight` to test different fixed fusion ratios.

## Testing

To test a saved checkpoint, run:

```bash
CUDA_VISIBLE_DEVICES=0 python main.py \
  --gpu 0 \
  --test True \
  --use_adaptive_rmts \
  --data asap \
  --model_name t5-base
```

The results will be saved under the corresponding result directory.
